# packages/proto

gRPC service contracts for Clustral, written in proto3 under the `clustral.v1` package.

| File | Service | Purpose |
|---|---|---|
| `cluster.proto` | `ClusterService` | Register, list, get, and deregister clusters |
| `tunnel.proto` | `TunnelService` | Bidirectional kubectl proxy stream (Agent ↔ ControlPlane) |
| `auth.proto` | `AuthService` | Issue and validate kubeconfig credentials; agent credential lifecycle |

---

## Regenerating .NET stubs

Stubs are generated by the `Grpc.Tools` MSBuild integration wired into
`packages/Clustral.Sdk`. You do **not** run `protoc` manually.

### One-time setup

Install the .NET tool manifest dependencies (already committed; only needed
on a fresh clone):

```bash
dotnet tool restore
```

### Regenerate after editing a `.proto` file

```bash
dotnet build packages/Clustral.Sdk
```

`Grpc.Tools` detects the changed `.proto` file(s) via MSBuild incremental
build and regenerates only what changed. Output lands in:

```
packages/Clustral.Sdk/obj/Debug/net9.0/  (intermediate, not committed)
```

The generated types are available immediately to any project that references
`Clustral.Sdk`.

> **Never edit files under `*/Generated/` or `*/obj/` by hand.** They are
> overwritten on every build.

### Confirming the generated surface

After building, inspect the generated files:

```bash
ls packages/Clustral.Sdk/obj/Debug/net9.0/*.cs
```

Or open the project in your IDE — generated types appear in the `Clustral.V1`
namespace.

## MSBuild wiring reference

The `Clustral.Sdk.csproj` should contain something like the following for each
`.proto` file (the `GrpcServices` attribute controls whether client stubs,
server stubs, or both are emitted):

```xml
<ItemGroup>
  <Protobuf Include="../../packages/proto/cluster.proto"
            GrpcServices="Both"
            ProtoRoot="../../packages/proto" />
  <Protobuf Include="../../packages/proto/tunnel.proto"
            GrpcServices="Both"
            ProtoRoot="../../packages/proto" />
  <Protobuf Include="../../packages/proto/auth.proto"
            GrpcServices="Both"
            ProtoRoot="../../packages/proto" />
</ItemGroup>
```

`GrpcServices="Client"` for `Clustral.Agent` (only needs the client side of
`TunnelService`); `GrpcServices="Server"` for `Clustral.ControlPlane` server
implementations; `GrpcServices="Both"` in the Sdk so consumers can choose.

## Evolving the contracts

Proto3 field numbers are **permanent**. Follow these rules to stay
wire-compatible:

- **Add** new fields freely — old clients ignore unknown fields.
- **Never reuse** a field number, even after removing a field. Use `reserved`
  instead:
  ```protobuf
  reserved 7;
  reserved "old_field_name";
  ```
- **Never rename or re-number** existing fields in a released version.
- **Add** new `oneof` variants or enum values; never remove or renumber them.
- Breaking changes require a new package version (`clustral.v2`) and a new
  `csharp_namespace` (`Clustral.V2`).
